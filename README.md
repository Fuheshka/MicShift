# MicShift 🎙️⚙️

**MicShift** — это легкая и мощная системная утилита для Windows на базе .NET 10, предназначенная для управления, переключения, заглушения и автоматического переключения микрофонов.

Главная фишка проекта — **автоматическое переключение микрофонов в зависимости от «расстояния»** (на основе интеллектуального сравнения уровней громкости с настольного микрофона и гарнитуры).

---

## 🌟 Основные возможности

1. **Автоматическое переключение (Auto-Distance):**
   * Работает в фоне на основе сравнения пиковых уровней сигнала двух микрофонов (настольного и гарнитурного).
   * **Алгоритм защиты (гистерезис):** Исключает ложные срабатывания от посторонних шумов и эха. Переключение происходит только в том случае, если уровень звука на активном устройстве превышает порог тишины ($> 3\%$) и стабильно держится выше другого микрофона в $1.5$ раза на протяжении $\approx 200$ мс (6 циклов опроса).
2. **Фоновый режим в системном трее (System Tray):**
   * При запуске в режиме трея консольное окно полностью скрывается.
   * Контекстное меню иконки трея позволяет быстро переключать дефолтный микрофон, включать/отключать автоматический режим, делать мут и закрывать программу.
3. **Глобальные горячие клавиши (Global Hotkeys):**
   * Работают глобально (даже во время полноэкранных игр):
     * `Ctrl + Alt + M` — быстро заглушить/включить звук активного микрофона (Mute/Unmute).
     * `Ctrl + Alt + S` — переключить дефолтный микрофон на следующее активное устройство по кругу (Cycle).
4. **Нативные Toast-уведомления:**
   * При переключении микрофона или заглушении звука в углу экрана всплывает нативное уведомление Windows с информацией о текущем статусе устройства.
5. **Интерактивный консольный интерфейс (Spectre.Console):**
   * Выбор устройств стрелочками и Enter, плавная индикация калибровки звука (VU-метры) без мерцания экрана.
6. **Автоматизация через CLI:**
   * Поддержка аргументов запуска для интеграции в скрипты автоматизации или Stream Deck.
7. **Логирование:**
   * Полное логгирование событий через Serilog в `%appdata%/MicShift/logs/`.

## 🛠️ Параметры командной строки (CLI)

* `--list` (или `-l`) — вывести список всех активных микрофонов с их GUID-идентификаторами и признаком устройства по умолчанию.
* `--default` (или `-d`) — показать текущее устройство связи по умолчанию.
* `--switch <Name or GUID>` (или `-s`) — быстро переключить микрофон на указанный (по GUID или фрагменту имени) и завершить работу.
* `--tray` (или `-t`) — запустить приложение сразу в фоновом режиме системного трея (консольное окно не отображается).
* `--help` (или `-h`) — показать справку.

Если запустить приложение без параметров, откроется **интерактивное консольное меню**.

## ⚙️ Первоначальная настройка и калибровка

Чтобы функция автоматического переключения по расстоянию работала корректно:
1. Запустите приложение в интерактивном режиме (без аргументов).
2. Выберите пункт **`[C] Calibration mode`**.
3. Выберите из списка ваш **настольный (desk)** микрофон, а затем **микрофон гарнитуры (headset)**.
4. Вы увидите шкалы громкости (VU-метры) для проверки работоспособности. Произнесите несколько слов в каждый из микрофонов. Нажмите `Q` для завершения калибровки.
5. Настройки сохранятся в файле `%appdata%/MicShift/settings.json`.
6. Теперь вы можете запустить режим трея (`[T]` из меню или флаг `--tray` при автозагрузке), и автопереключение начнет работать в фоновом режиме!

## 📦 Сборка проекта

Для сборки проекта в виде единого исполняемого файла (`Single File Self-Contained Exe`), который содержит в себе все зависимости и не требует установленного .NET на компьютере пользователя:

```bash
dotnet publish MicShift.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:PublishReadyToRun=true -p:PublishTrimmed=false -o ./publish
```

Исполняемый файл `MicShift.exe` появится в папке `./publish`.
